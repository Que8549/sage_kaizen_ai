# Sage Kaizen Philosophy

## Overview

Sage Kaizen is designed as a dual-mode intelligence architecture integrated with physical computing systems and real-time visual interfaces.

It aligns:

- High-performance GPU inference
- Raspberry Pi distributed agents
- Real-world LED visualization
- Voice interaction loops
- Continuous improvement methodology

---

# The Dual-Brain Model

## Fast Brain (RTX 5080)

Purpose:
- Low latency responses
- Default conversational routing
- Creative drafting
- Lightweight reasoning
- Tool invocation routing

Optimized for:
- Speed
- Efficiency
- 16K context when feasible
- GPU isolation

Represents:
Operational intelligence.

---

## Architect Brain (RTX 5090)

Purpose:
- Long-context reasoning
- System design
- Deep debugging
- Documentation generation
- RAG synthesis
- Tutor-level explanations
- Architectural refactoring

Optimized for:
- Maximum GPU layer offload
- Larger context windows
- Accuracy over latency

Represents:
Sage.

# Engineering Discipline

Sage Kaizen follows invariant rules:

- .bat files are configuration only
- EXE invoked directly
- Always use --log-file
- Fully expanded paths
- No delayed expansion assumptions

Architecture without discipline regresses.
Kaizen prevents regression.
